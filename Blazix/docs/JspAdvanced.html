    Forms Processing

    If you have processed user input from HTML forms in JSP, you will appreciate the changes Blazix brings!

    Blazix takes advantage of JSP 1.1 "standard tag library" mechanism to add a Blazix tag library.  There are only two tags added by Blazix tag library for forms processing, but just these two tags can cut down your forms processing workload by more than half!  See the Blazix tag library for details on how to use the Blazix tag library and details on the forms processing tags.

    SQL

    SQL is often used in conjunction with JSPs.  In addition to the forms processing tags, Blazix also includes a short and effective set of SQL tags to make SQL writing an easy task.

    Email

    Sending email from a JSP page is very simple with Blazix.  Simple email, HTML email or email with attachments can all be handled very easily.  See Blazix tag library for details .

    Internationalization

    Just because JSP makes it so easy to embed Java code directly inside HTML, internationalization can become much more complicated!  Blazix tag library also includes tags for internationalization.  These do require a bit more work for tagging the pieces of language to translate, but this little bit of extra work saves serious troubles at internationalization time.

    Reserved Variable Names

    Blazix reserves variables name starting with _ds or _desisoft or _blazix and attribute name strings starting with "desisoft." or "blazix.".  These prefixes should not be used in development.

    WSP

    Blazix also makes the JSP technology available for writing WML, for wireless devices.  You can just write normal WML pages, rename them with the extension .wsp, and embed Java in them as with the .jsp pages.  The server will take care of the book-keeping.  It will track the different versions of index.jsp, index.wsp etc. The wireless visitors will be automatically routed to index.wml and index.wsp instead of index.html and index.jsp.  And if any situations require a message from the server, the message will go in approprite WML instead of in HTML.
