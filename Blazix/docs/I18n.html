    Internationalizing web-sites using Blazix

    Blazix supports internationalization of static and dynamic web pages.

    Static Pages

    For static web pages such as HTML, GIF, JPEG pages whose content is delivered from a file, multiple language versions of the file can be created.

    The language specific version of a file is created by adding two underscores and the language code before the extension in the filename.

    For instance, if the file is index.html and if the language is french, the language specific file will be named index__fr.html.

    Two underscores are used to avoid accidental name collision.

    This mechanism is also available for use with other static files such as GIF files or JPEG files.

    JSP pages

    Internationalization in JSPs is supported via a JSP 1.1 tag library.  See details in Blazix JSP for Advanced Users.

    Note that the Java code in JSPs still should be written to be internationalization aware.  For instance, to format a date, instead of using the default DateFormat instance, a locale specific DateFormat instance should be obtained, using the request.getLocale() method to obtain the visitor's locale.

    Other dynamic content

    Servlets must handle internationalization in the servlet code, using the getLocale() method to obtain the visitor's locale.
