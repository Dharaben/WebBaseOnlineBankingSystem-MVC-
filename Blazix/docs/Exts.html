    The .wsp and .xsp extensions

    Blazix includes support for utilizing the JSP technology for WML and XML files.

    The .wsp extension is enabled in Blazix to work similar to the .jsp extension, but with the default mime-type of WML instead of HTML.

    Similarly, the .xsp extension provides the mime-type text/xml.

    Since XML is intended to provide machine-processable data, Blazix can easily provide dynamically generated XML content to any user agents that are seeking to accept XML.

    With WML, there is further Blazix support in the form of appropriate forming of any error messages (such as exceptions.)

    Note about .wsp files:  In .wsp files, the page should not be started with a JSP directive, but it should be started with the standard <?xml syntax.  This is because JSP directives on a line are replaced by an empty line as per JSP specifications.  Therefore if the <?xml is preceded by a line containing a JSP directive, it will not be the first item in the file.

    JSP directives can be added after the first line.
