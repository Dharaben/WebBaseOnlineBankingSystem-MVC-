    desisoft.deploy.AuthCheck  interface

    This interface is used to add authorization to web-server URLs and EJBs.  The interface is defined as follows: 
    public interface AuthCheck
{
    void initialize( java.util.Hashtable props );
    boolean authorize(
              String appname,
              String username,
              String password,
              javax.servlet.ServletRequest request );
    boolean isUserInRole(
              String appname,
              String username,
              String role );
    boolean isUserInAnyRole(
              String appname,
              String username,
              String[] roles );
}
    The "initialize" method is passed any initialization parameters from the configuration file.

    The "appname" refers to the application or EJB bean name.  The "authorize" method must return true to indicate authorization succeeded, false otherwise.  The "isUserInRole" and "isUserInAnyRole" functions are called after authorization has succeeded, to check valid roles for the authorized user.  The "username" parameter may be null to indicate no username was supplied.

    The "request" parameter to "authorize" is null when validating EJBs.  When validating website visitors, it can be used to work with the request, e.g. to retrieve other additional data from a database about the user who is logging in and place the additional data in the session.
