    JSP for Beginners

    For beginners learning JSP, we recommend the JSP tutorial at http://www.jsptut.com/

    There are some server specific items that you need to know:

    As you go through the JSP tutorial, the JSP files will be placed in C:\Blazix\webfiles folder (unless you have modified the default configuration.)  The URL for the top level of this folder is http://DESKTOP-NKM5MK3/   E.g. the file C:\Blazix\webfiles\test\hello.jsp, would correspond to the URL http://DESKTOP-NKM5MK3/test/hello.jsp in your browser.

    The JSP directory is for holding temporary JSP files, not for classes used by the JSP files.  If your JSP is using any classes (e.g. a bean,) you can place the class files in the C:\Blazix\classes folder.  You can also place it in any folder in your CLASSPATH.  If you have to change the CLASSPATH, make sure to stop and restart the web server so it can pick up the new CLASSPATH.  (Depending upon your system, you may also have to reboot the machine or log out and log back in so your new CLASSPATH is in effect.)  Jar files cannot be added to the C:\Blazix\classes folder, they must be added to the CLASSPATH.

    After learning JSP, if you find you need to debug a JSP in Blazix, there is a "jspdebug" command available, that you can use to generate debuggable Java files.

    Blazix also has the ability to internationalize your JSPs easily, so you can write JSPs that will be seen differently by visitors with different language preferences.

    Blazix enables the ".wsp" extension for WML files, so you can take advantage of JSP technology for creating dynamic pages for wireless devices.
