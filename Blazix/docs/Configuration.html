    Server Configuration

    Blazix servers each use a separate configuration file.  There is a configuration file for the web-server, one for the EJB server, and one for the cluster backup servers.  There is also one for the Windows NT service.

    The installation process adds a sample configuration file web.ini for the web-server, and ejb.ini for the EJB server.

    All configuration files contain lines of the form 

    property-name: value
    where property-name typically consists of three dot-separated parts: the type of the resource, a name for the resource and the property for which a value is being specified.  For instance: 
    dataSource.myJdbc1.url:  myUrl
dataSource.myJdbc1.class: myClass
    Here a "dataSource" resource is being configured.  "myJdbc1" is a user-supplied name.  The "url" property of the dataSource is being configured, and it is being set to the value "myUrl".  The "class" property of the dataSource is being set to "myClass".

    Before a user-supplied name such as myJdbc1 can be used in a configuratin file, it must be "registered" using a ".name" line, e.g 

    dataSource.name: myJdbc1
    There will often be several such name registrations, e.g. 
    dataSource.name: myJdbc1
dataSource.name: myJdbc2
application.name: app1
application.name: app2
    Properties which do not require a user-supplied name may have one or two dot-separated parts instead of three.

    In most cases, the user-supplied name is just for internal use within the configuration file.  E.g. by specifying a name for a dataSource, the user can specify parameters for various data sources.

    Comments:  Lines starting with # or ; in a configuration file are considered comments.
