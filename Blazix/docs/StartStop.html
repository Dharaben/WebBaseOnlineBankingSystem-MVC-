    Starting and Stopping the Servers

    Starting the servers

    There are many different ways to start the servers.
    • The Windows installer adds commands to the "Start" menu to start the web server and the EJB server. 
    • The blazix commands or their Java class names can be used to start the web server, the EJB server or the clustering server.
    • For production and clustered environments, Blazix servers should be started using the server manager.

    Stopping the Servers

    Like in any standard Java programs, typing a control-C in the Java console window will stop the servers.  A disadvantage of this approach is that log files will not get flushed, and connections will not be gracefully closed.  It should not be used in production environments.

    The server manager should be used in production environments.  If the server manager is in use, servers can be stopped and restarted either locally or on remote machines, using the Blazix server administration interface.
