    The Blazix tag library:  Internationalization tags

    Translation Overview

    The Blazix internationalization tags make it possible to mark the natural-language parts of the JSPs, provide multiple translations for the natural-language parts, and have the appropriate translation be delivered at run-time.

    There are two tags: <blx:xltFile> and <blx:xlt>.

    The entire text which is to be translated should be placed inside <blx:xltFile> and </blx:xltFile>.  This tag provides a "base name" for the translation file.  There is no need to have separate base names for separate JSP pages, the entire site could have a single set of translation files.  Or more, as convenient.

    A "language suffix" is added to "base name" to obtain the actual translation file name.  For instance, if the translation file is "translationFile", and if the current user's language is german, then the filename which will be used will be "translationFile_de", because the language code for german is "de".  The language codes can be further narrowed by specifying a country code and a variant, as per Java Locale conventions.  The narrowest filename which is available, will be used.  Therefore if the user's language code is "en_uk" and a "translationFile_en_uk" is available, that file will be used.  Otherwise, if a "translationFile_en" is available, that will be used.  (Note:  The translation tool has language codes for a wide number of languages, built it.)

    If no translation file is available, the original text will be used.

    In addtion the language suffix to the base name, extensions are preserved.  If any file extension (starting with the dot charcter) is present, the language suffix will be added before the dot.  So if the base file is "translate.txt" and the language code is "fr", the resulting filename will be "translate_fr.txt".

    The <blx:xlt> tag is used to tag the English (or other natural language) text which is to be translated.

    An example is: 

    <blx:xltFile file="translate.txt">
<P><blx:xlt ref="intro">
Introduction to
this site..
</blx:xlt><br>
<blx:xlt ref="getname">Enter your name here</blx:xlt>
<FORM METHOD=POST ACTION=savename.jsp>
<INPUT NAME=name TYPE=TEXT>
<INPUT TYPE=SUBMIT VALUE=<blx:xlt ref="submit">Submit</blx:xlt>>
</FORM>
</blx:xltFile>
    The translation file format is 
    tag=value
    with one translation per line.  Lines starting with the dash ("-") character are considered continuation of the previous line (the dash will be replaced by line separator characters.)

    For instance, the translation file for french for the above example would be named translate_fr.txt and it might contain the following text 

    intro=Introduction à
-ce site
getname=écrivez votre nom ici
submit=Soumettent
    Use the Blazix Tag Extractor tool to collect all translation tags from your JSP files and give them to translators for individual languge translation.

    Individual XLT tags are described below.

    blx:xltFile

    This tag can be used to build internationalizable JSPs.
        
    The name provided by the "file" attribute is  the "base" name of the translation file.  A locale specific extension is then added to pull in   the appropriate language translation file.  For instance if the base name is "myfile" and the language is "fr", the translation file will be "myfile_fr".  If the file has an extension, the language id is added before the extension.  E.g.  for "myfile.txt" and language id "fr", the tag library will look to see if "myfile_fr.txt" exists.
        
    Attributes:
     
    Attribute Name Attribute Description Required
    file Base name of translation file yes

    blx:xlt

    The text which should be translated is placed inside this tag.  The tag itself has a a reference id "ref".  This reference id is then used by translators to provide other language translations.
        
    At runtime, the translation for the visitor's language is pulled from the translation files using the "id" attribute.
        
    All occurences of this tag must be inside an "xlatFile" tag.
        
    Attributes:
     
    Attribute Name Attribute Description Required
    ref Tag reference name yes
