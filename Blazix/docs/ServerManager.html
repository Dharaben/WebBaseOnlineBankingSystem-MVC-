    Cluster Administration in Blazix

    The Blazix Server Administration tool can  be used to remotely shutdown and start servers, view log files, view or upload configurations, check status, and deploy or redeploy EJB Jars and WEB archive (WAR) files.

    To take advantage of these facilities, the servers should be setup to be managed by the Blazix Server Manager.  The server manager runs at each node in the cluster, and manages all Blazix servers (web servers, EJB servers, JMS servers, cluster backup servers) at that node.  Only one copy of the server manager needs to be run at the node.

    On Windows systems (NT or 2000), the server manager can be started via the provided Blazix Windows Service program.  This program should be configured and installed as a service.

    For non-Windows systems, the server manager should be started via the machine startup sequence (the "RC" files in typical Unix systems.)  The command file (shell file or batch file) that does this should change directory to the Blazix installation directory, it should then set the classpath, and then it should invoke the server manager via the

    java desisoft.tools.ServerManager
    command.

    In both cases, Windows or other systems, a configuration file for the server manager is required.  In the case of Windows, the NT (or Windows 2000) service's ".ini" file also serves as the server manager's configuration file, so a separate file does not need to be written -- instead the configuration commands should be placed in the service's ".ini" file.

    On the machine from where the servers will be managed (i.e. from where the Server Administration tool will be run), the cluster.ini file must contain configuration information for each Server Manager on the cluster.

    For further details, see the Blazix Windows Service or the Server Manager configuration.
