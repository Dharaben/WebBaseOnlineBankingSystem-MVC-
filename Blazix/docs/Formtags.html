    The Blazix tag library:  Form processing tags

    The tags available for forms processiong are blx:setProperty and blx:getProperty

    To work with multiple-selection list boxes using these tags, provide getter/setter methods that return/accept "String[]", to retrieve or set an array of values.

    blx:setProperty

    This tag is similar to jsp:setProperty, with improved handling for empty strings and exceptions.

    A method may be named via the onError attribute  to handle errors and continue processing.  Empty values entered  by users reset the bean fields.  (For this to work with checkboxes, list all checkboxes in the checkbox attribute.)  Set stringNull attribute to true if empty values should reset string objects to null, false if empty values should reset string objects to the empty string.

    Attributes:
     
    Attribute Name Attribute Description Required
    name Name of bean yes
    property Bean property to set.  Use * to set all properties. yes
    onError Name of a method which will be called upon an exception.  The method may be a public method declared in the JSP page itself.  Otherwise, it must be a public static method, and its fully qualified name must be specified in the tag.  If this field is not specified, exceptions will be throw to the JSP page, terminating tag processing.  The method signature is
      public void onError( String fieldname,
                           String value,
                           Exception exception );    		 no
    stringNull Indicates whether to set strings to null or to the empty string when the form sends an empty field.  Set to true or false. no
    checkbox Lists names of all the checkbox fields in the form.  If the checkboxes are not listed in this attribute, they will not be reset because the browsers do not send checkbox field names.
    Separate multiple checkbox field names by commas.    		 no
    emptyInt Specifies an "empty" value for the integers.  If the form sends an empty field, the integers are set to this value.  By default, the empty value is 0. no

    blx:getProperty

    This tag is the inverse of jsp:setProperty or blx:setProperty.

    Instead of retriving form fields into a bean, it places data from a bean into an outgoing form.

    The input fields which are being modified  must be enclosed beteen <blx:getProperty> and </blx:getProperty>.  The entire form may be enclosed between a single tag instance, or multiple tag instances may be used to enclose multiple input fields.

    This tag works on the INPUT, SELECT and TEXTAREA fields.

    Attributes:
     
    Attrribute Name Attribute Description Required
    name Name of bean yes
    property Property to use.  * for all properties. yes
    dateFormat Describes how any outgoing date objects should be formatted.  Should be short, medium, long or full. no
    timeFormat Describes how any outgoing date objects should be formatted.  Should be short, medium, long or full. no
    emptyInt Defines an "empty" value for integers.  If the bean contains this value, the form will be set to the empty string.  By default, there is no empty value.

    To format date objects as date only, specify a value for dateFormat only.  To format date objects as time only, specify a value for timeFormat only.

    Example:

    The following example assumes that the bean "testBean" has a method named "getValue()".  The method will be called and the value returned from it wil be placed in the <INPUT> tag. 

    <blx:getProperty name=testBean property=* dateFormat="short">
<FORM METHOD=POST ACTION=target.jsp>
Value <INPUT NAME=value TYPE=TEXT><BR>
<INPUT TYPE=SUBMIT>
</FORM>
</blx:getProperty>
