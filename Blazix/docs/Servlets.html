    Configuring servlets in Blazix

    Servlets are specified in the web server configuration file.
    A typical servlet configuration looks like: 
    servlet.name: myservlet
servlet.myservlet.className: mypackage.MyServletClass
servlet.myservlet.url: /mysrvlet
    A "servlet.name" line must be added for each servlet, and the servlet must be given a name for the web-server configuration file.  This name is then used to specify other properties, namely a class which implements the servlet, and a URL at which the servlet will be available.

    The above configuration will make the servlet available at /mysrvlet and will use the class mypackage.MyServletClass for the servlet implementation.

    The servlet class must be available in the CLASSPATH, or it may be placed in the C:\Blazix\classes folder, or in a folder configured as "server.classDir" in the web configuration file.

    After a new servlet entry has been added to the configuration file, the web server should be stopped and restarted.  Servlets can also be added to a running server via the server administration tool, but this requires placing them in a web archive (WAR) file.  (For development environments and given the quick startup time of Blazix, during development it may be simpler just to stop and restart the server!)

    If the servlet class file has changed, it is not necessary to stop and start the server.  The default web server configuration file provided with the installation has already configured the URL
    http://DESKTOP-NKM5MK3/_reload
    as a URL for reloading all classes in the default application.  Just visit this URL after the servlet classes have changed.
