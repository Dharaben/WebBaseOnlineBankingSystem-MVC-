    desisoft.dsap.Ion class

    This class is used by Ion clients to look up interfaces provided by Ion servers.  It contains many variations of the "lookup" method, which is used to lookup an interface at a remote location.  The lookup may be done by specifying the interface class itself, or by specifying an agreed-upon URL string (not related to the web URLs.  The ION URL is simply a string that the server and the client agree upon.)

    The class also contains a "close" method which can be used to explicitly close the connection.

    If server callbacks are to be used, a callback object can be provided via the "setCallback" method.  If the call requires authorization, a username and password can be provided via the "setAuthorization" method. 

    public class Ion
{
    // Methods to lookup an interface.
    public Object lookup( String host, Class intf ) throws IOException;
    public Object lookup( String host, String url ) throws IOException;
    public Object lookup( String host, String url, Class intf ) throws IOException;
    public Object lookup( String host, int port, Class intf  ) throws IOException;
    public Object lookup( String host, int port, String url  ) throws IOException;
    public Object lookup( String host, int port, String url, Class intf  ) throws IOException;
    public Object lookup( InetAddress host, int port, Class intf ) throws IOException;
    public Object lookup( InetAddress host, int port, String url ) throws IOException;
    public Object lookup( InetAddress host, int port, String url, Class intf ) throws IOException;

    // Method to close connection.
    public void close();

    // Method to specify callback object.  If required, call BEFORE calling "lookup".
    public void setCallback( Object c, Class intf );

    // Method to specify username/pwd.  If required, call BEFORE calling "lookup".
    public void setAuthorization( String user, String pass );
}
    The port, if not specified, defaults to 80.  If the URL and interface both are specified, both must match.

    Sample usage is:
      Ion ion = new Ion();
      MyInterface myintf =
        (MyInterface) ion.lookup( "www.myhost.com", MyInterface.class );
      // Use the inteface
      // ...
      // ...
      // Close when done, for better resource usage.
      // Otherwise connection will take up resources until
      // the ION object gets garbage collected.
      ion.close();

    For SSL connections, use the desisoft.dsap.IonSsl class.  This class has the identical behavior except that it uses SSL, and that the port defaults to the default SSL port of 443 if it is not specified.
