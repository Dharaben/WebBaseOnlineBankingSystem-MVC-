    What is new in Blazix 1.1 and 1.2

    This page is for Blazix users who are familiar with Blazix 1.0 or 1.1, and simply want a quick overview of what is new in Blazix.

    1.2
    Servlets 2.3:  Blazix 1.2 catches up to the latest release of servlets.  This version of servlets introduces various things, most useful of which may be filters.  Full details on servlets 2.3 are available at various sites on the web.

    JSP 1.2:  Also added in Blazix 1.2  There are not that many new and useful features in the latest JSP release, but some developers may find the XML oriented features interesting.

    1.1
    ION:  This is a new technology available in Blazix, that makes it simple to do distributed processing.  The server and client agree upon a Java interface, the server registers an object to implement the interface and the client looks up and uses the interface.

    JMS:  Java Messaging Service.

    E-mail:  The JSP tag library included in Blazix (BLX) now has support for email, including the ability to send attachments.

    Cluster Management:  Now a cluster of servers can all be easily remotely managed, with uploads, log file access, configuration, startup and shutdown all available remotely.

    Class Reloads:  Blazix developers have frequently asked for this, so now Blazix allows the ability to reload Java classes without having to restart the server.  This includes full support for WAR file deployment and redeployment, but for development purposes a simple URL is provided that may be entered in the browser, or bookmarked.

    Automatic EJB Primary Keys:  EJB primary keys can now be automatically generated, as an added convenience.

    Virtual Hosting:  Blazix servers can now be used for virtual hosting of multiple domain names at a single IP address.  See web-server configuration parameters for how to set this up.
